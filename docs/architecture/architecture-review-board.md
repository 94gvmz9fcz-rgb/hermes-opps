# Architecture Review Board

## Purpose
Prevent Hermes from drifting into unnecessary complexity or disconnected one-off tools.

## Responsibilities
- Receive external LLM feedback
- Challenge proposals
- Accept, reject, or defer recommendations
- Record reasoning
- Protect roadmap sequencing
- Prevent architectural drift

## Decision Types
- Accepted
- Partially Accepted
- Deferred
- Rejected

## Rule
No material architecture change is adopted until reviewed and recorded here.
